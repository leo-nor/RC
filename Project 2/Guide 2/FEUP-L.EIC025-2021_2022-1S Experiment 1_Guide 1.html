Experiment 1/Guide 1

In this lab class we will configure IP addresses and connect to a switch leading to the following network:

Y is the bench number.

See below on the Helper section for examples.

Steps

  1. Disconnect the switch from netlab and connect tux computers to it.
  2. Configure tuxy3 and tuxy4 to have an IP address and have the network 172.16.Y0.0/24 defined
    • using ip addr and ip route commands
    • you can also use respectively ifconfig and route
  3. Take note of the IP and MAC addresses of the network interfaces on both tuxes.
  4. Use ping command to verify connectivity between these computers.
    • Note that there is no name resolution for the IP addresses defined, so you need to use the IP addresses.
    • You can have a look a /etc/hosts.
  5. Inspect the forwarding table (ip route show or route –n) and ARP (ip neigh or arp –a) tables
  6. Delete ARP table entries in tuxy3 (ip neigh del <ipaddress> or arp –d <ipaddress>)
  7. Start Wireshark in eth0 of tuxy3 and start capturing packets
  8. In tuxy3, ping tuxy4 for a few seconds
  9. Stop capturing packets
  10. Save log and study it at home to answer the questions below.

Helpers

Both iproute2 and net-tools are in production. iproute2 contains the new command line programs  and will eventually replace net-tools. Below we will have both options. You can see the "equivalence" in, for example, iproute2 vs net-tools in LinOxide.

How to reinitialize the network system: 
root# /etc/init.d/networking restart
or 
root# systemctl restart networking

Configuring the network:

All commands for configuring the network should be given as root. The indication for it is root# or just #

Viewing configurations does not usually require root privileges.

net-tools iproute2
link up/activate eth0
 
# ifconfig eth0 up
 
# ip link set up eth0
list current net itfs' configurations 
# ifconfig
 
# ip link show
# ip addr show
or 
# ip l s
# ip a s
configure eth0 with IP address 192.168.0.1 and netmask of 16 bits 
# ifconfig eth0 192.168.0.1/16
 
# ip addr add 192.168.0.1/16 dev eth0
or 
# ip a a 192.168.0.1/16 dev eth0
add a route to the subnet 
# route add -net 192.168.1.0/24 gw 172.16.4.254 eth0
 
# ip route add 192.168.0.1/24 via 172.16.4.254 dev eth0
add a default route 
# route add default gw 192.168.1.1 eth0
 
# ip route add default via 192.168.1.1 dev eth0
list current routes 
# route -n
 
# ip route
list ARP entries 
# arp -a
 
# ip neigh
delete an ARP entry 
# arp -d <ipaddress>
 
# ip neigh <ipaddress>

Questions

  • What are the ARP packets and what are they used for?
  • What are the MAC and IP addresses of ARP packets and why?
  • What packets does the ping command generate?
  • What are the MAC and IP addresses of the ping packets?
  • How to determine if a receiving Ethernet frame is ARP, IP, ICMP?
  • How to determine the length of a receiving frame?
  • What is the loopback interface and why is it important?
